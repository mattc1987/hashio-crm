# Apps Script backend

Tiny web app that lets the Hashio CRM read & write the backing Google Sheet.

Deploy in 5 minutes — see [`../SETUP.md`](../SETUP.md) for the click-by-click guide.

## Files

- `Code.gs` — paste this into a fresh Apps Script project bound to the CRM Google Sheet.

## Endpoints

All are GET. Every request requires `key=<your-api-key>`.

| Action | Query | Description |
|---|---|---|
| `ping` | `?action=ping&key=…` | Health check |
| `read` | `?action=read&tab=Deals&key=…` | Read one tab |
| `readAll` | `?action=readAll&key=…` | Read all CRM tabs in one call |
| `write` | `?action=write&payload=<json>&key=…` | Create / update / delete a row |

Write payload:

```json
{
  "entity": "deals",
  "op": "update",
  "payload": { "id": "dl1", "stage": "Closed Won" }
}
```

## Script properties

- `API_KEY` — shared secret between the frontend and this script. Generated by running `setupApiKey()` once from the editor.
- `SHEET_ID` (optional) — if the script is NOT bound to the sheet, set this to the sheet ID.
